# 验证清单与验收标准 (VERIFY.md)

> 本文档定义了功能上线前的“生死门禁”。

## 1. 业务功能验收 (UAT)

| 场景编号 | 场景描述 | 期望结果 (Expected) |
| :--- | :--- | :--- |
| **TC-001** | 正常下单全链路。 | 订单状态变为 `PAID`，仓库收到通知。 |
| **TC-002** | 重复支付防御。 | 第二次支付请求报错并路由至历史对账单。 |
| **TC-003** | 库存零边界测试。 | 下单失败，库存保持 0 不变。 |

---

## 2. 性能底线 (Performance)

- [ ] 核心查询平均响应时间 < 200ms。
- [ ] 批量下单事务耗时 < 500ms（引用: 商户端极致体验计划）。

---

## 3. 安全与合规性

- [ ] 关键接口具备防重刷频率限制。
- [ ] 所有外键关联及敏感参数必须经过权限校验。
- [ ] 隐私数据在展示时必须已执行脱敏。
