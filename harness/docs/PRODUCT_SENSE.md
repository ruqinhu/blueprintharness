# 产品感知 (PRODUCT_SENSE.md)

> 文档权威等级：P2
> 本文档定义了业务的核心价值主张、场景地图及高风险点。

## 1. 业务价值画布

| 核心问题 | 描述 |
| :--- | :--- |
| **目标用户** | 中小企业采购员。 |
| **核心痛点** | 审批流多、下单繁琐、发票管理混乱。 |
| **核心功能** | 快捷下单、自动审批流、一键开票。 |

---

## 2. 核心业务地图 (Business Map)

- **Map A: 基础链路** (注册 -> 选品 -> 下单 -> 支付 -> 配送)
- **Map B: 营销链路** (满减优惠券 -> 积分抵扣 -> 秒杀)
- **Map C: 后台管理** (商品上架 -> 订单核销 -> 财务对账)

---

## 3. ⚠️ 高风险点清单 (High-Risk)

| 风险环节 | 风险描述 | 防护建议 |
| :--- | :--- | :--- |
| **金额计算** | 优惠叠加导致的负数或计算精度偏差。 | 引入 Big Decimal 库并配置多层审计（Reference: BR-003）。 |
| **并发超卖** | 抢购期间的库存竞争。 | 使用 Redis Lua 脚本或乐观锁。 |
| **外部支付** | 支付回调丢失或重复通知。 | 实现支付幂等性校验。 |
| **隐私数据** | 用户收货地址与手机号泄露。 | 数据库加密存储。 |
