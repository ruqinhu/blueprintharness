# Blueprint Harness — 任务执行工作流 (Workflow)

本文定义了 AI 与人类开发过程中必须遵守的阶段性门禁（HARD-GATE）与任务原子化标准。

## 1. HARD-GATE 工作流 (阶段门禁)

**禁止跳步。** 只有完成当前 Phase 的产出并获得批准后，方可进入下一 Phase。

| 阶段 | 动作 | 准入/准出条件 |
| :--- | :--- | :--- |
| **Phase 0: 启动** | 阅读 `harness/spec/` 下的所有核心文档 + `memory/lessons.md` + `workspace/` 下的相关日志。 | **准入**：任务启动。**准出**：在对话中明确列出你识别出的 1-3 个核心业务点。 |
| **Phase 1: 理解** | 基于 `harness/tasks/specs/_TEMPLATE.md` 新建规格书。 | **准入**：Phase 0 完成。**准出**：人类批准位于 `harness/tasks/specs/` 下的具名规格书文件。 |
| **Phase 2: 分解** | 生成原子任务计划 `harness/tasks/active/`。 | **准入**：Phase 1 完成。涉及“相关经验模式”扫描。**准出**：人类批准任务列表。 |
| **Phase 3: 实现** | 编写代码，严格遵守 `@blueprint-ref` 锚点引用。 | **准入**：Phase 2 完成。**准出**：代码编写完成，测试通过。 |
| **Phase 4: 验证** | 运行验证脚本，回答“防呆双问”。 | **准入**：Phase 3 完成。**准出**：验证脚本提示 PASS 且已回答任务末尾的 [防呆双问](#2-防呆双问定义)。 |
| **Phase 5: 归档** | 迁移任务至 `completed/`，沉淀经验至 `memory/`。 | **准入**：Phase 4 完成。**准出**：目录清理完毕。 |

---

## 2. 防呆双问定义

在 Phase 5 归档任务前，必须在任务文件末尾回答以下两个核心问题：
*   **Q1: 本次实现与业务文档有出入吗？**（评估偏差及风险）
*   **Q2: 本次是否发现了可复用的模式？**（提取知识至 `lessons.md`）

---

## 3. 原子任务标准 (Atomicity)

*   **规模控制**：单次 Task 的规模应严格控制在整个特性开发预估工作量的 **1/5 ~ 1/10**。
*   **语义完整**：每个任务必须是语义完整的单元。若发现实现路径过长，必须进一步拆分。
*   **可追溯性**：每个任务必须关联 `harness/tasks/specs/` 中定义的 AC（验收准则）。

---

## 4. 连续性日志 (Workspace Journaling)

在每个 Session 结束或重大决策后，建议在 `harness/workspace/[user]/journal.md` 中记录：
*   **决策背景**：为什么选择方案 A 而非方案 B。
*   **遇到的坑**：当前的蓝图在哪个环节定义模糊或有冲突。
*   **待办提醒**：下一个 Session 需要注意的特殊状态扭转逻辑。
